# Mycelium Ink 任务拆分与开发计划

**版本**：v1.0  
**日期**：2026-05-29  
**总周期**：6 周（MVP v1.0）  
**目标**：完成可商用的网页端 + 桌面端基础版本

## 1. 项目总体 Roadmap

| 阶段 | 周期 | 目标 | 交付物 | 验收标准 |
|------|------|------|--------|----------|
| **M1 算法原型** | Week 1 | 验证核心创意 | 可交互 HTML Demo | 天气+音频实时驱动书法+菌落 |
| **M2 网页版 MVP** | Week 2-3 | 完整网页可用版 | PWA 在线版本 | 满足 PRD 1.4 核心指标 |

## 2. 详细任务拆分（按周）

### **Week 1：M1 算法原型（核心验证）**

**目标**：快速验证“天气→书法 + 音频→菌落”是否好看

- [ ] 项目初始化（Monorepo + Vite + TypeScript + Core 包）
- [ ] 实现 Weather Service + Open-Meteo 接入 + 缓存机制
- [ ] 完成 Brush Engine（书法生成）基础版
  - 词库 + 随机内容生成
  - 气象参数映射 + 真实运笔动画
  - 宣纸纹理（Simplex Noise）
- [ ] 完成简单 Colony Agent 系统（1000 粒子）
  - 有限状态机（FSM）
  - 基本生长、分裂、死亡逻辑
- [ ] 基础双层渲染（Canvas 2D + WebGL）
- [ ] 手动控制面板（滑块 + 麦克风输入）
- [ ] 简单合成与导出 PNG

**里程碑交付**：一个可本地运行的 `index.html` Demo，能实时响应天气和麦克风。

---

### **Week 2：网页端基础功能**

- [ ] 完整 UI 实现（右侧悬浮面板 + 4 个 Tab）
- [ ] 音频分析器优化（AudioWorklet + FFT + 特征提取）
- [ ] 高级书法效果（飞白、晕染、压力模拟）
- [ ] 菌落系统增强（趋墨性、寻路、配色切换）
- [ ] 时间侵蚀引擎基础（氧化、代谢）
- [ ] PWA 配置 + Service Worker
- [ ] 响应式布局 + 移动端适配
- [ ] 导出功能（4K PNG + 参数水印）

---

### **Week 3：网页端完善 & 性能优化**

- [ ] 动态 LOD + 性能监控系统
- [ ] WebGL 降级方案实现
- [ ] 层积书写（每小时新字）
- [ ] 历史沉积记录回放
- [ ] 完整错误处理与降级流程（天气、手动模式）
- [ ] 性能测试（低端设备）
- [ ] UI 动效抛光（水墨感过渡）

**M2 里程碑**：网页版 PWA 可公开发布

### **Week 4：桌面端开发（Electron）**

- [ ] Electron 项目搭建 + 主进程 / 渲染进程分离
- [ ] 系统音频捕获（desktopCapturer）
- [ ] 平台适配层实现（AudioSource 抽象）
- [ ] 桌面端专属功能（安装包、托盘、自动更新基础）
- [ ] 离线能力增强（本地 JSON 缓存）
- [ ] 打包配置（体积优化 < 180MB）
- [ ] 跨平台测试（Win / macOS）

### **Week 5：时间侵蚀 & 稳定性打磨**

- [ ] 延时摄影录制功能（桌面端优先）
- [ ] 24 小时稳定性测试
- [ ] UI/UX 最终抛光（毛玻璃、留白、暗色模式）
- [ ] Logo + 品牌资产集成

- [ ] 内测收集反馈（目标 10+ 用户）
- [ ] 完整文档（README、用户手册、开发者指引）

- [ ] 风险项最终验证

## 3. 任务优先级总表（关键路径）

**Must Have（MVP 核心）**：
1. 天气映射 → 书法生成
2. 音频分析 → 菌落生长
3. 双层渲染合成
4. 手动控制面板
5. Open-Meteo + 降级机制
6. 4K 导出
7. 系统音频（桌面端）

**Should Have**：
- 时间侵蚀与层积
- 历史回放
- 性能 LOD
- PWA + Electron 打包

**Could Have**（v1.1 后再做）：
- LLM 诗句生成
- MIDI 输入
- 延时摄影网页端实现
- 更多菌落生态

---

## 4. 资源与人力假设

- **主力开发者**：1 人（全职）
- **设计师**：兼职或外包（UI 动效）
- **测试**：开发者自测 + 内测用户
- **每日建议投入**：6-8 小时（避免烧尽）

---

## 5. 风险与缓冲

- **最高风险**：菌落系统性能（分配最多缓冲时间）
- **次高风险**：Electron 系统音频稳定性
- **缓冲**：每阶段预留 1 天缓冲时间

---

## 6. 跟踪与管理建议

- **工具**：Linear / Notion / GitHub Projects
- **每日站会**：记录进度与阻塞
- **周会**：周五复盘 + 下周计划
- **代码规范**：ESLint + Prettier + 核心模块单测覆盖率 > 70%
- **版本控制**：`main` / `develop` + feature 分支
